Five Minute Grids

by @mickmacks

Introduction

CSS grids provide browser-wide* built-in grid functionality that can replace the need for loading bootstrap, foundation, or materialize grid systems. CSS Grids are handled using three main properties — Grids, Rows & Columns, and Gaps. Working in conjunction, these properties create grid templates that we can populate with content.

CSS Grids allow us to easily align, justify and order content within our grid

Grids

Parent elements (darker blue) are used to contain and structure child elements (lighter blue) in rows and columns. The first example uses "display: grid;", turning it's children into block level elements while the second two use "display: inline-grid", with the child items only taking as much space as needed.

Rows & Columns

Once a parent grid is set up, we can begin specifying rules for our rows and columns using "grid-template-row" or "grid-template-column". We can also now begin to use "fr", a fractional unit, to do percentage based math divisions for us.

Have you ever split something up evenly between X number of people, only to realize you miscounted? Or the number later changed? Fractional units let us instantaneously correct portion sizes without having to do the math.

grid-template-rows: 50px 100px;

grid-template-columns: 50px 100px;

Gaps

As you may have guessed, once we have our grid structure and child elements sized how we want them, the last main component is gaps (also known as gutters in typography). This is pretty simple, you just specify what you want the overall grid gaps to be, or specify just the rows or columns.

grid-gap: 2rem;

grid-row-gap: 2rem;

grid-column-gap: 2rem;
